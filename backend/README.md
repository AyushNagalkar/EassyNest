## Description

[Nest](https://github.com/nestjs/nest) framework TypeScript starter repository.

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```

## Deployment

When you're ready to deploy your NestJS application to production, there are some key steps you can take to ensure it runs as efficiently as possible. Check out the [deployment documentation](https://docs.nestjs.com/deployment) for more information.

If you are looking for a cloud-based platform to deploy your NestJS application, check out [Mau](https://mau.nestjs.com), our official platform for deploying NestJS applications on AWS. Mau makes deployment straightforward and fast, requiring just a few simple steps:

```bash
$ npm install -g @nestjs/mau
$ mau deploy
```

With Mau, you can deploy your application in just a few clicks, allowing you to focus on building features rather than managing infrastructure.

## Troubleshooting & Redis Gotchas

### ⚠️ Upstash Redis Connection Timeout (Port 63799)
If you see the following warning/error in the console logs:
```
[ioredis] Unhandled error event: Error: connect ETIMEDOUT
Redis connection failed — refresh token revocation disabled
```
* **Why it happens**: Your local internet service provider (ISP) or router firewall is blocking outgoing TCP traffic on non-standard ports (like `63799` used by Upstash Redis).
* **Self-Healing Fallback**: The backend scoring engine is built with a resilient fallback mechanism. If the connection to the BullMQ queue fails or times out (after 2.5s), the request is intercepted, and the system **synchronously runs the rule-based matching logic**, writes it to the database with `source: RULE_BASED`, and returns the results. Your application will keep working 100% fine.
* **How to fix this locally**:
  1. Check your network firewall rules and allow outgoing TCP traffic on port `63799`.
  2. Alternatively, install a local Redis instance on the default port (`6379`) and set:
     ```env
     UPSTASH_REDIS_URL="redis://localhost:6379"
     ```
